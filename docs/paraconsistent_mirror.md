# ZeroGateSim Paraconsistent Conflict-Locality Mirror

**Line:** `v1.3.2-alpha`  
**Status:** projection mirror only

This mirror does not claim that ZeroGateSim is Priest logic or any complete paraconsistent logic system.

It asks one narrower question:

> When positive-looking pressure and contrary witness coexist, does the conflict stay local instead of exploding into arbitrary final +1?

## Native boundary

ZeroGateSim's native witness still decides final output.

The paraconsistent mirror reads after the native final-output layer. It does not alter:

- distinction;
- polarity;
- relation;
- observed return;
- `C_Z = min(D, P, R, B)`;
- raw expression;
- return-depth;
- temporal lineage;
- echo-independence;
- earned-one.

## Core rule

```text
raw +1 plus debt must not explode into arbitrary final +1
```

A candidate may carry both positive-looking pressure and contrary witness. That is useful pressure. It is not permission to crown.

## Projection values

- `conflict_localized` — Belnap-both conflict pressure was held or demoted instead of crowned.
- `conflict_overcrowned` — Belnap-both conflict pressure became final +1; this is a locality breach.
- `true_without_conflict` — positive evidence without contrary witness.
- `false_without_conflict` — contrary witness without positive expression pressure.
- `neither_without_conflict` — no decisive pressure in this mirror.

## Outputs

Matrix runs write:

```text
matrix_paraconsistent_mirror_summary.csv
matrix_paraconsistent_mirror_read.md
```

## What this mirror preserves

It preserves the local/nonlocal shape of conflict pressure:

```text
candidate has positive-looking pressure
candidate also has contrary witness
conflict remains local
final +1 is not automatically granted
```

## What this mirror loses

It does not preserve the whole native ZeroGate stack. It cannot replace:

- return-depth;
- temporal lineage;
- echo-independence;
- role witness;
- final trinary output.

It also does not prove that contradiction is metaphysically fundamental.

## Success posture

A useful run can say:

```text
Conflict pressure appeared.
It remained local.
No final +1 was granted merely because the candidate had positive-looking pressure.
```

## Failure posture

A breach looks like:

```text
Belnap-both conflict pressure became final +1.
```

That does not disprove the whole project. It marks a specific locality wound to inspect.
