## Why this exists

`v1.6.11-alpha` compresses the planned roadmap-truth audit and feature-design proposal into one gate.

The reason is simple: the project had begun to react one wound at a time. That is dangerous in a research repo. The route must state what the next several steps are, why each gate exists, and what would stop the line.

This release does not implement new shadow features. It repairs the map and defines what the next feature implementation is allowed to attempt.

## Feature design proposal

The next implementation should add observable, role-stripped feature columns. These are not targets and must be computed before target loading.

| lane | feature family | purpose |
|---|---|---|
| relation-specific | relation ownership / borrowed-relation contrast | distinguish owned relation from field echo and borrowed coherence |
| return-specific | return integrity / collapse-vs-return contrast | distinguish real return-with-memory from zero crossing or collapse |
| demotion | demotion trajectory / refusal pressure | distinguish why pressure was refused, not merely that pressure was high |
| zero-hold ambiguity | structured zero / not-yet pressure | preserve zero as witness instead of failure bucket |
| density residual | residual pressure after dumb baselines | keep the one useful pressure signal while refusing to crown it as detector |

Forbidden shortcuts:

```text
truth_role
trap / expresser labels
target false-one rates
family names or row order
post-score targets
manual score retuning after seeing outcome
```
