# ZeroGateSim Cross-Logic Comparison Presets

**Introduced:** `v1.4.1-alpha`  
**Truth repair:** `v1.4.2-alpha`  
**Status:** run-plan generator, not evidence

The cross-logic comparison report introduced in `v1.4.0-alpha` reads completed matrix runs. This file explains the next small step: writing repeatable run plans so stronger comparisons are not hand-assembled every time.

A preset is not proof. It is a recipe for producing local matrix outputs that can later be read by `zerogate_sim.cross_logic_report`.

## Why this exists

The first v1.4 report can read one or more matrix directories, but without a preset the operator has to remember which candidate profiles and weather profiles belong together.

The preset writer gives a stable, inspectable plan:

```powershell
$env:PYTHONPATH = (Join-Path (Get-Location) "src")
& $P -m zerogate_sim.comparison_preset --preset adversary_triad27 --out runs\comparison_preset_plan_v1_4_2
```

It writes:

```text
comparison_preset_read.md
comparison_preset_manifest.csv
run_preset.ps1
```

The human reads `run_preset.ps1` before execution. The script runs matrix commands and then runs the cross-logic comparison report.

## v1.4.2 path truth repair

Generated preset scripts now check the exact cross-logic report path before building the assistant handoff. If the report is missing, the script fails instead of producing a green-looking bundle without the report.

The script also prints the exact handoff ZIP path to upload.

This repairs the earlier operator wound where a manual follow-up command looked for:

```text
runs\comparison_preset_adversary_triad27\cross_logic_comparison\cross_logic_comparison_read.md
```

but the generated preset report actually belonged under:

```text
runs\cross_logic_presets\adversary_triad27\cross_logic_report\cross_logic_comparison_read.md
```

The preset script is now the path authority. Do not invent the report path by hand.

## Presets

### `quick_smoke`

A one-run wiring check.

Use when:

- checking the preset writer;
- checking cross-logic report command shape;
- avoiding heavier run cost.

### `adversary_triad27`

A small three-run comparison across the three proof wounds:

```text
adversary_distinction
adversary_polarity
adversary_relation
```

Each run uses `triad27` weather with a small seed count. This is the first useful stronger comparison preset.

### `wide_adversary_probe`

A heavier three-run comparison using `wide243` weather for the same three adversarial corpora.

Use only when local runtime cost is acceptable.

## Boundary

The preset layer writes commands. It does not execute them automatically.

Allowed claim:

> The preset creates a repeatable plan for generating stronger cross-logic comparison evidence.

Forbidden claim:

> The preset proves ZeroGateSim is stronger, physical, or equivalent to known logic systems.

## Generated artifacts

Preset outputs belong under local `runs/` folders and should not be committed.

The useful upload after execution is normally an `assistant_test_handoff.zip` generated by the plan.

## Operating sentence

A preset is a map, not the territory. Run outputs are evidence; the plan is only the route.
