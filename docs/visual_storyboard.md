# Visual Storyboard

This page explains the visual language used for ZeroGateSim public-facing diagrams.

The accepted format is simple:

- off-white background;
- large readable labels;
- trinary color logic;
- thick directed flow;
- no decorative geometry that does not carry mechanism;
- no mystery arrows.

The visuals should teach the witness path before they decorate the theory.

## Figure 2 — Intended Trinary Witness Stack

File: `docs/assets/trinary_witness_stack.svg`

Purpose: show why raw +1 should not become final truth automatically. This is a
design target, not an exact v1.7.11 implementation trace.

Mechanism:

The intended design sends raw local expression through:

- truth role;
- temporal lineage;
- echo-independence.

Current correction: truth role and echo-independence affect the role-aware
verdict; temporal lineage is emitted beside it and is not consumed.

Final output becomes:

- +1 earned-one;
- 0 structured witness hold;
- −1 rejection / false-one demotion.

DREED pressure: this prevents readers from saying the system merely renamed failure. The zero-state is structured and active.

## Rule

Every visual must answer one question:

> What pressure does this help the reader witness?

If a visual only looks good and does not carry pressure, it returns to zero.
