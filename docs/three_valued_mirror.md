# ZeroGateSim Kleene / Lukasiewicz Compression Mirror

**Line:** `v1.3.3-alpha`
**Status:** projection mirror only

This mirror is not an identity claim.

ZeroGateSim is not Kleene K3 logic and not Lukasiewicz L3 logic. The mirror only asks what happens when native final trinary output is compressed into a three-valued form:

```text
+1 earned-one -> T / true
0 witness-hold / latent / debt / wound -> U / unknown
-1 resist / reject / demotion -> F / false
```

## Why this mirror comes after fuzzy, Belnap, and paraconsistent mirrors

The earlier mirrors preserve more of the native machinery:

- the fuzzy mirror reads continuous gate pressure;
- the Belnap mirror separates evidence-for and evidence-against final +1;
- the paraconsistent mirror checks whether conflict pressure stays local.

The Kleene / Lukasiewicz compression mirror is deliberately harsher. It collapses the zero-zone into one middle value so we can measure what disappears.

## What is preserved

The mirror preserves only the broad final posture:

- accepted expression;
- held / unknown pressure;
- resisted or rejected pressure.

## What is lost

The mirror cannot preserve:

- `0+` fertile hold;
- `0` witness hold;
- `0-` quarantine hold;
- latent containment;
- latent overcrown demotion;
- relation debt;
- expresser wound;
- return-depth;
- temporal lineage;
- echo-independence;
- role-aware adversarial witness.

This loss is not a bug. It is the reason for the mirror.

## What the matrix writes

When a matrix run finishes, ZeroGateSim writes:

```text
matrix_three_valued_mirror_summary.csv
matrix_three_valued_mirror_read.md
```

The summary records the three-valued projection for every final-output candidate and marks zero-compression loss.

## Proper interpretation

Allowed:

> The three-valued mirror shows how much native ZeroGate zero-state information is lost when final output is compressed to true / unknown / false.

Forbidden:

> ZeroGateSim is equivalent to Kleene or Lukasiewicz logic.

Forbidden:

> Unknown means the same thing as native zero.

Native zero is an active witness zone. The three-valued unknown is only a compressed projection.

## Operating sentence

Three-valued compression is useful because it fails loudly: it shows what the native zero-zone refuses to flatten.
