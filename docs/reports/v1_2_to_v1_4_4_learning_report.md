## Claim boundary

This report does not add a new proof record and does not change the claim boundary.

ZeroGateSim remains a simulation-supported research-alpha framework. The supported claim remains bounded: inside generated synthetic fields, the zero-gate / earned-one witness can be tested for its ability to separate earned expression from raw expression pressure, latent overcrown, relation or return debt, and false-one pressure.

Unsupported claims remain unsupported:

- no proof of physical dimensional genesis;
- no proof of cosmology;
- no proof that reality itself is trinary;
- no role-blind false-one discovery yet;
- no equivalence with fuzzy, Belnap, paraconsistent, Kleene, Lukasiewicz, or other known logic systems.

This report records what the v1.2-v1.4.4 line established, what it clarified, and what evidence is required next.

## Executive summary

The v1.2-v1.4.4 line moved ZeroGateSim from a first-research-alpha proof floor into a more disciplined experimental framework.

The core movement was:

```text
native math lock
-> invariant tests
```

The important scientific lesson is not that the system became bigger. It became harder for the system to lie politely.

1. **Native first.** External logic families are comparison mirrors, not identity claims.
2. **Weakest gate matters.** `C_Z = min(D, P, R, B)` remains the native coherence floor.

4. **Evidence handling is part of the science.** A report, preset, or handoff that loses evidence is not a harmless tooling issue; it changes what can be trusted.

## v1.2 — Native math lock

The v1.2 line made the native mechanism explicit and testable.

Core native anchors:

```text
E0 = (Z0, tau)
T3[X](tau)
Li = (-ei, 0, +ei)
Gamma_i = D_i P_i R_i
C_Z = min(D, P, R, B)
chi_raw
Z(0)..Z(4)
chi_earned
```

The crucial distinction is:

```text
Gamma = D * P * R
```

versus:

```text
C_Z = min(D, P, R, B)
```

`Gamma` is return-potential generated by distinction, polarity, and relation. It is not the full coherence gate. The full zero-gate witness also requires observed return `B`. This distinction must remain protected in tests and documentation.

The v1.2 lesson:

> A theory that cannot trace its own equation into code and invariant tests is not ready for stronger claims.

## v1.3 — Known-logic mirrors

The v1.3 line added known-logic comparison mirrors.

These mirrors do not define ZeroGateSim. They expose what is preserved, distorted, or lost when native ZeroGateSim states are projected into familiar formal surfaces.

### Fuzzy / many-valued mirror

Purpose:

```text
Compare native weakest-gate coherence against continuous aggregation surfaces.
```

Useful diagnostic:

```text
average_overcrown_pressure
```

This catches cases where an average score looks acceptable while one native gate remains weak. The lesson is that continuous scoring can be useful pressure, but averaging can hide a wounded gate.

### Belnap evidence-state mirror

Purpose:

```text
Represent evidence-for, evidence-against, both, and neither.
```

This is useful when raw positive-looking pressure coexists with contrary witness. A Belnap-style `both` state is not permission to crown. It is a conflict-pressure witness.

### Paraconsistent conflict-locality mirror

Purpose:

```text
Check whether conflict pressure stays local instead of exploding into arbitrary +1.
```

The question is narrow: if positive-looking pressure and contrary witness coexist, does the system contain the contradiction or crown through it?

### Kleene / Lukasiewicz compression mirror

Purpose:

```text
Show what is lost when native final output compresses to true / unknown / false.
```

The central loss is zero-state richness. Native `0+`, `0`, `0-`, relation debt, latent overcrown, quarantine, and witness hold cannot all be faithfully represented by one flat unknown.

The v1.3 lesson:

> Known logics are mirrors. They are useful only when they make a native wound easier to see. They are not borrowed authority.

## What was learned scientifically

### 1. Raw expression is not enough

The project repeatedly reinforced the central repair:

```text
raw expression is pressure
earned-one is final +1
```

A candidate may look locally expressible and still deserve hold, quarantine, or demotion.

### 2. Zero is active

The zero-state is not a failure bucket. It can preserve:

- latent overcrown;
- relation debt;
- return debt;
- quarantine pressure;
- witness hold;
- not-yet expression.

This is one of the most important differences between the native witness and flat binary reporting.

### 3. Return is the survival witness

Distinction can make a candidate visible. Polarity can make it axis-like. Relation can make it field-bound. Return asks whether it comes back through zero with memory, continuity, and persistence.

Return is therefore not an optional fourth label. It is the gate that prevents beautiful local structure from becoming premature final one.

### 4. Mirrors report loss

The known-logic mirrors are valuable because they show different kinds of loss:

- fuzzy averaging can hide a weak gate;
- Belnap states can reveal unresolved evidence conflict;
- paraconsistent projection can test conflict locality;
- three-valued compression can erase native zero-state detail.

### 5. Evidence transport matters

The v1.4 handoff and preset work showed that scientific software needs trustworthy evidence movement, not just green unit tests. Generated reports must be present, uniquely preserved, and traceable to their source run folders.

## What remains unproven

The following remain open:

- role-blind false-one detection;
- threshold robustness;
- ablation necessity for each witness layer;
- survival under independent generator families;
- comparison against external structure measures;
- external reproduction by another person or machine;
- any physical or cosmological bridge.

The current line supports continued experimentation. It does not support overclaiming.

## External-review readiness condition

The project should not be presented to a serious external reviewer as a theory of reality. It should be presented only when the package is tight enough to respect their time.

Minimum readiness package:

- one-page claim boundary;
- one-page mechanism summary;
- one reproducible command;
- one seed-block result table;
- one ablation table;
- one failure-mode section;
- one question for the reviewer.

Best external-review question:

```text
Does this operational distinction between raw expression and earned expression resemble any known formal, physical, or information-theoretic structure, and what would be the strongest falsification or comparison target?
```
