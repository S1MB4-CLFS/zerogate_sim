# Temporal lineage witness

Status: v0.2.12-alpha

Temporal endurance asks whether a candidate kept its name after time touched it.
Temporal lineage asks how it moved while time touched it.

The witness reads early / witness / late posture as a path:

- expression endured
- matured to expression
- expression returned to zero
- late fertile hold
- late witness hold
- late quarantine hold
- decayed to rejection
- rejected through time
- mixed lineage

This is not a gate mutation. It is a path-reading layer.

The core law remains unchanged: +1 expression is still earned by strength and
zero-gate coherence. A lineage path does not crown a candidate. It tells the
human witness whether the candidate endured, matured, returned, quarantined, or
failed.

New outputs:

- `matrix_lineage_read.md`
- `matrix_lineage_transitions.csv`
- `matrix_lineage_candidate_summary.csv`

## Wide243 profile

The `wide243` matrix adds a fifth trinary axis: temporal depth.

- time-minus: compressed temporal field
- time-zero: baseline temporal field
- time-plus: stretched temporal field

This creates 3^5 = 243 scenario cells.

## Candidate corpus triad27

The default corpus remains `alpha12`.

The wider corpus `triad27` expands the test field to F00..F26. It is not used by
default because changing scenario weather and candidate weather at the same time
can hide mechanism pressure. Use it deliberately when testing broader geometry.
