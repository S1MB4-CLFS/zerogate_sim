# Zero-State Taxonomy

**Version:** v0.2.10-alpha  
**Scope:** witness-layer refinement, not gate-engine mutation

Zero is not one flat state.

In v0.2.9, ZeroGateSim learned to separate expression into `+1`, `0`, and `-1`. That was necessary, but not enough. A single `0 held_latent` bucket mixed together near-expression, genuine uncertainty, and unsafe fog.

v0.2.10 splits the 0-state into a trinary zero taxonomy:

- `0+ fertile_hold` — near expression; retest before repair temptation.
- `0 witness_hold` — genuine ambiguity; hold and observe.
- `0- quarantine_hold` — unsafe latent pressure; do not count and do not build on it.

The complete witness grammar is:

- `+1 expressed` — earned one.
- `0+ fertile_hold` — not one yet, but close.
- `0 witness_hold` — hold at zero for more pressure.
- `0- quarantine_hold` — latent but unsafe.
- `-1 rejected` — failed active witness.

This update does not loosen the gate. It changes the language of what happens before expression.

A real one is still what zero can return as without lying.

The primate may want every non-expression to be failure. The witness refuses. Some things are almost born. Some things need weather. Some things need quarantine.
