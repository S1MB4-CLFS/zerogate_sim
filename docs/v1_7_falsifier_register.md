# v1.7 Falsifier Register

**Introduced:** `v1.7.0-alpha`  
**Purpose:** name what can break the core question before more evidence work begins.

## Core question

> Can a final trinary witness distinguish earned-one from raw expression pressure, latent overcrown, relation debt, return debt, and false-one pressure under controlled synthetic-field adversarial weather?

## Falsifiers

| falsifier | breaks the claim if | minimal repair |
|---|---|---|
| lane visibility failure | a required lane is missing, collapsed into generic hold, or explainable only by role label | lane taxonomy repair or core question narrowing |
| earned-one dead-safe failure | zero false crowns happen only because real earned-one is refused | compare against dead-safe baseline and report earned-one preservation |
| false-one final crown | any designed trap or false-one pressure receives final `+1` | stop closeout, inspect gate and final-witness path |
| return specificity collapse | return-potential, observed return, return-depth, and return debt blur into one vague return score | run the v1.7.1 Return Gate Trace Lock |
| baseline equivalence | raw, binary, dead-safe, average-gate, no-return, no-relation, no-lineage, no-echo, or no-zero-hold witnesses explain the result equally well | baseline and ablation falsifier matrix |
| role-label recounting | lane assignment is just designed-role recounting | masked role-dependence audit |
| holdout failure | the pattern survives only the reference profile or known seed family | fresh holdout synthetic-field challenge |
| reviewer path failure | a cold reviewer needs private chat history to understand the claim and evidence | reviewer-start-here package |

## Overdo warning

Do not add more machinery just because a falsifier is uncomfortable. The smallest honest repair wins. The bigger repair can wait its turn in the queue like a civilized goblin.
