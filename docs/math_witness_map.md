# ZeroGateSim Native Math Witness Map

**Line:** `v1.2-alpha`  
**Purpose:** lock the native geometry → math → code → test chain before external logic comparison.

This document is not a new theory layer. It is a witness map. Its job is to prevent ZeroGateSim from racing into Kleene, Łukasiewicz, Priest, Belnap, or fuzzy-logic comparison before the native mathematical object is explicit.

The working discipline is simple:

> No known-logic mirror before native math witness.

## Claim boundary

ZeroGateSim remains a generated toy-field proof-of-concept. It does not prove cosmology, physical dimensional genesis, or that reality itself is trinary.

The supported claim remains bounded:

> Inside generated toy fields, a zero-gate / earned-one witness can separate earned expression from raw expression, latent overcrown, and false-one pressure under distinction, polarity, and relation adversarial tests.

The current v1.2 addition is narrower:

> The repository now makes its native geometry, equations, implementation locations, invariants, and future logic-mirror relevance explicit.

## How to read this map

Each native object is recorded in six lanes:

1. **Geometry** — the shape-intuition.
2. **Equation** — the formal operator or measurement rule.
3. **Code trace** — where the current implementation lives.
4. **Invariant** — what tests must protect.
5. **External-logic relevance** — how this may later be compared.
6. **Overclaim boundary** — what must not be said.
